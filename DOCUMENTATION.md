## 1. Project Overview

**AI Music Studio** is a full-stack web application that brings generative AI to music creation. Users can:

- Type a text description and receive an AI-composed instrumental track in seconds.
- Generate multi-language lyrics (English, Telugu, Hindi, Tamil, mixed) powered by Groq LLaMA.
- Remix existing audio by uploading a file and describing a new style.
- Auto-generate duration-matched background music for uploaded videos.
- Identify songs by humming or singing via AudD audio fingerprinting.
- Explore and play community-created tracks in a public gallery.
- Compete in an AI-powered music trivia quiz.

The project is built on **Flask + Meta MusicGen + Groq LLaMA 3.3 70B** and is deployable in one click to Render.com.

## 2. Features

| # | Feature | Description |
|---|---------|-------------|
| 1 | 🎵 **Text-to-Music Generation** | Describe any sound in plain text; MusicGen produces a WAV file up to 3 minutes long. |
| 2 | 🎤 **Lyrics Generator** | Groq LLaMA writes complete structured lyrics (verse/chorus/bridge) in English, Telugu, Hindi, Tamil, or mixed Tanglish style. |

| 5 | 🔍 **Song Discovery (Hum)** | Record yourself humming or singing and identify the song via AudD audio fingerprinting (with Groq AI fallback). |
| 6 | 📝 **Lyric Search** | Paste a lyric fragment; Groq AI returns the top 3 matching songs across all languages. |
| 7 | 🌐 **Community Gallery** | Browse and play public tracks generated by all registered users. |
| 8 | ✨ **Prompt Enhancer** | AI rewrites a casual description into a professional MusicGen-optimised prompt (BPM, key, genre, mood, instruments). |
| 9 | 🎙️ **Voice Input** | Speak your music description into the microphone; Google Speech Recognition transcribes it into the prompt field. |
| 10 | 🎮 **Music Quiz** | A randomised 5-question music trivia game powered by a built-in question bank + 2 AI-generated questions per round. |
| 11 | 👤 **User Profiles** | Per-user track history, play counts, and download stats. |
| 12 | 🔐 **OTP Authentication** | Passwordless login via email or phone; OTP is printed to console in development mode. |

## 5. Architecture & Data Flow

### Music Generation Flow

```
User types prompt
      │
      ▼
POST /api/generate-music
      │
      ├─ _build_prompt()  ← adds instrument / mixer level modifiers
      │
      ├─ model.generate([prompt])  ← MusicGen-Small on CPU/GPU
      │
      ├─ audio_write() → static/generated/<timestamp>.wav
      │
      ├─ groq_call()  ← AI quality score (0–100) + feedback
      │
      ├─ save_track()  → SQLite
      │
      └─ JSON response → Frontend plays WAV via WaveSurfer.js
```

### Remix Flow

```
User uploads audio + new prompt
      │
      ▼
POST /api/generate-music  (multipart form with `file` field)
      │
      ├─ torchaudio.load(uploaded_file)
      │
      ├─ model.generate_continuation(waveform, sample_rate, [prompt])
      │
      └─ Same save + response as above
```

### Video BGM Flow

```
User uploads video + mood description
      │
      ▼
POST /api/video-bgm
      │
      ├─ VideoFileClip(video).duration  ← extract duration (capped at 300 s)
      │
      ├─ _build_prompt("background music for a video, <mood>…")
      │
      ├─ model.set_generation_params(duration=extracted_duration)
      │
      └─ model.generate([prompt])  →  save  →  JSON response
```

### Song Discovery Flow (Hum Search)

```
User records humming
      │
      ▼
POST /api/discovery/voice-search
      │
      ├─ pydub: normalize + boost (+6 dB) + mono + 44.1 kHz
      │
      ├─ AudD API  ← audio fingerprinting (like Shazam)
      │         │
      │    ┌────┴────────────────────────────────┐
      │    │ Matched                             │ No match
      │    ▼                                     ▼
      │  Return song + Spotify URL    Google STT → Groq AI suggest top 3
      │
      └─ JSON response → frontend
```

## 6. Database Schema

The SQLite database is stored at `instances/studio.db` and is initialised automatically on first run via `init_db()`.

### `users` table

| Column | Type | Description |
|--------|------|-------------|
| `id` | TEXT (PK) | User's email or phone number |
| `name` | TEXT | Display name |
| `otp` | TEXT | Current one-time password (NULL after verification) |
| `created_at` | TEXT | ISO datetime, defaults to `datetime('now')` |

### `tracks` table

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER (PK, AUTOINCREMENT) | Track ID |
| `user_id` | TEXT (FK → users.id) | Owner's email/phone |
| `title` | TEXT | Auto-generated title from prompt |
| `artist` | TEXT | User's display name at time of generation |
| `filename` | TEXT | WAV filename in `static/generated/` |
| `audio_url` | TEXT | Flask `url_for` path to the WAV |
| `prompt` | TEXT | Original text prompt used to generate |
| `duration` | INTEGER | Track length in seconds |
| `plays` | INTEGER | Play count (default 0) |
| `is_public` | INTEGER | 1 = visible in gallery, 0 = private |
| `created_at` | TEXT | ISO datetime |

## 7. API Reference

All protected endpoints require an active session (login). Unauthenticated requests are redirected to `/login`.

### 7.1 Authentication APIs

#### `POST /api/auth/send-code`
Sends (prints in dev) a 4-digit OTP and creates or updates the user record.

**Request body (JSON)**
```json
{ "identifier": "user@example.com" }
```

**Response**
```json
{
  "success": true,
  "dev_otp": "4821",
  "is_returning": false
}
```

---

#### `POST /api/auth/verify`
Verifies the OTP and creates an authenticated session.

**Request body (JSON)**
```json
{ "identifier": "user@example.com", "otp": "4821" }
```

**Response**
```json
{
  "success": true,
  "user": { "name": "New Artist", "is_returning": true }
}
```

---

#### `GET /api/auth/logout`
Clears the session and redirects to `/`.

---

#### `POST /api/auth/update-profile`  *(login required)*
Updates the logged-in user's display name.

**Request body (JSON)**
```json
{ "name": "DJ Sai" }
```

---

#### `POST /api/auth/delete-account`  *(login required)*
Permanently deletes the account and all associated tracks.

## 10. Installation & Local Setup

### Prerequisites
- Python 3.11+
- `ffmpeg` installed and on PATH (required by pydub / MoviePy)
- A Groq API key (free at [console.groq.com](https://console.groq.com))

### Steps

```bash
# 1. Clone the repository
git clone https://github.com/saibhanush/AI-Music-Studio-project-.git
cd AI-Music-Studio-project-

# 2. Create and activate a virtual environment
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Set environment variables
export GROQ_API_KEY=your_groq_api_key_here
export AUDD_API_TOKEN=your_audd_token_here    # optional

# 5. Start the development server
python app.py
```

Navigate to `http://localhost:5000`.  
Sign up with any email or phone number — the 4-digit OTP is printed to your terminal.

> ⚠️ **MusicGen downloads ~1 GB model weights** on first run from Hugging Face. Ensure you have a stable internet connection and ~4 GB RAM.

## 11. Deployment on Render.com

### One-click deploy

1. **Fork** this repository to your GitHub account.
2. Go to [render.com](https://render.com) → **New** → **Web Service** → connect your forked repo.
3. Render auto-detects `render.yaml`. Set the following environment variables in the Render dashboard:
   - `GROQ_API_KEY`
   - `AUDD_API_TOKEN` *(optional)*
4. Click **Deploy**.

Render runs `render-build.sh` (install deps) then starts `gunicorn app:app` automatically.

### render.yaml summary

```yaml
services:
  - type: web
    name: ai-music-studio
    env: python
    pythonVersion: 3.11
    buildCommand: "chmod +x ./render-build.sh && ./render-build.sh"
    startCommand: "gunicorn app:app"
    envVars:
      - key: GROQ_API_KEY
      - key: AUDD_API_TOKEN
      - key: XFORMERS_DISABLED   # value: "1"
      - key: NO_XFORMERS         # value: "1"
```

> ⚠️ **MusicGen requires ~4 GB RAM.** Use at least a **Standard** Render plan. The free tier will run out of memory.

## 12. AI Models Used

### Meta MusicGen-Small
| Property | Value |
|----------|-------|
| Creator | Meta AI Research |
| Parameters | 300 million |
| Architecture | Auto-regressive Transformer (EnCodec tokens) |
| Training data | 20,000 hours of licensed music |
| Max duration | 300 seconds (5 minutes) |
| FAD score | 7.8 |
| MOS score | 3.8 / 5.0 |
| Released | June 2023 |
| Hugging Face | `facebook/musicgen-small` |

MusicGen is loaded **once at startup** and kept in memory for fast inference. It runs on CPU if no CUDA GPU is available.

**Generation parameters (defaults)**
```
duration:    20 s (overridden per request)
top_k:       250
top_p:       0.0
temperature: 1.0
cfg_coef:    3.0
```

### Groq LLaMA 3.3 70B Versatile
Used for:
- Lyric generation (multi-language)
- Prompt enhancement
- Quality scoring of generated tracks
- Song identification fallback
- Music quiz question generation

API calls are handled by the shared `groq_call(prompt)` helper (`max_tokens=1024`, `temperature=0.7`).

## 13. Frontend Overview

### Pages and their JS responsibilities

| Page | Key JS behaviour |
|------|-----------------|
| `showcase.html` | Polls `/api/public-stats` to display live counters |
| `login.html` | Two-step OTP form; shows/hides steps, calls `/api/auth/send-code` then `/api/auth/verify` |
| `index.html` (Studio) | Tab-based UI: Generate / Remix / Video BGM / Lyrics. Calls generation APIs, renders WaveSurfer waveform, download button |
| `discovery.html` | MediaRecorder for hum capture, lyric text search, renders results cards |
| `gallery.html` | Fetches `/api/gallery`, renders track cards with inline WaveSurfer players |
| `profile.html` | Loads stats and history; delete account modal |

### WaveSurfer.js
Every generated audio track is rendered as an interactive waveform using [WaveSurfer.js](https://wavesurfer.xyz/). Clicking the waveform seeks to that position; a play/pause button toggles playback.

### Voice Input
The browser's `MediaRecorder` API records `.webm` audio which is POSTed to `/api/voice-input`. Google Speech Recognition returns a transcript that is auto-filled into the prompt field.

## 14. Security & Session Management

| Setting | Value | Purpose |
|---------|-------|---------|
| `app.secret_key` | `pitti_guru_shahini_studio_2026` | Flask session signing |
| `SESSION_PERMANENT` | `True` | Sessions survive browser restart |
| `PERMANENT_SESSION_LIFETIME` | 7 days (604800 s) | Auto-logout after 1 week of inactivity |
| `SESSION_COOKIE_SAMESITE` | `None` | Required for cross-site embed scenarios |
| `SESSION_COOKIE_SECURE` | `True` | Cookie only sent over HTTPS |

> ⚠️ **For production use**, replace the hardcoded `secret_key` with a randomly-generated value stored as an environment variable.

### OTP Authentication
- OTP is a 4-digit integer generated with `random.randint(1000, 9999)`.
- Stored in the `users.otp` column and set to `NULL` after successful verification.
- In development the OTP is printed to stdout (no email/SMS gateway is integrated).

### File Upload Safety
- All uploaded filenames are sanitised with `werkzeug.utils.secure_filename`.
- Uploaded files are stored temporarily in the `uploads/` directory and deleted after processing.

## 15. Known Limitations

| Limitation | Details |
|------------|---------|
| **CPU generation is slow** | MusicGen-Small on CPU takes ~2–5 minutes for a 30 s clip. Use a GPU-enabled Render plan or run locally with CUDA. |
| **Max duration 300 s** | MusicGen caps generation at 300 seconds regardless of input. |
| **No email/SMS OTP delivery** | OTPs are only printed to the server console; integrate Twilio/SendGrid for production. |
| **No persistent file storage** | Generated `.wav` files are stored on the local filesystem. On Render's ephemeral storage they are lost on restart. Use cloud storage (S3/Cloudflare R2) for production. |
| **AudD API needed for hum search** | Without `AUDD_API_TOKEN`, hum-to-search falls back to Groq AI, which is less accurate. |
| **Secret key hardcoded** | Move `app.secret_key` to an environment variable before going to production. |
